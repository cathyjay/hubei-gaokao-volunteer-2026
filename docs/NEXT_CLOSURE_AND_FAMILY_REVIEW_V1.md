## 工作簿怎么看

1. `00_摘要`：本轮入口的规模、门禁和输出清单。
2. `01_第一闭环页列核验包`：37 个页列，仍按 E0、E1、E2 执行泳道排序。
3. `02_第一闭环64小包`：把 206 条任务按 `页码版面×动作桶` 拆成 64 个小包，每包最多约 10 条任务，适合并行核验。
4. `03_第一闭环任务状态`：206 条字段任务的公开状态机，不保存候选值、人工读数或私有 OCR 文本。
5. `04_下一轮55组讨论`：55 个重点专业组的家庭讨论入口，标记先讨论、先核页、先核限制或先核费用。
6. `05_重点组完整专业`：458 条组内专业明细，默认一行一个招生专业，用于判断能否服从调剂。

## 保真边界

- 第 19 期 PDF 原页或纸质原页仍是省招办原件底座。
- 湖北官方系统或省招办计划在可得时用于最终代码和字段确认。
- 高校官网、招生章程、官网 API、XLSX、PDF 或图片计划只用于 double check、补缺和发现冲突。
- OCR、机器坐标、第三方平台和自动抽取结果只作为线索，不直接写回字段事实。
- 公开仓库只保存计数、状态、证据编号和哈希；字段读数、候选值、截图、OCR 原文和人工备注留在 Git 忽略的私有工作台。
