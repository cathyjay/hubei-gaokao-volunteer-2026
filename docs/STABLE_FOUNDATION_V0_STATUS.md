# 稳定数据基座 V0 状态

更新时间：2026-06-28

## 当前结论

稳定数据基座 V0 已经可以用于“候选发现”和“核验优先级排序”，但不能直接作为定稿依据、字段写回依据或湖北官方计划替代品。

这意味着：现在不应继续等待全量 13,736 条逐专业明细都人工核完后才开始筛学校专业。正确路径是先用 V0 底座筛出候选专业组，再对入围专业组做 100% 字段、完整专业组、调剂和章程限制核验。

## 为什么可以进入下一步

我们的目标不是做一个全省招生计划数据库产品，而是在 2026-06-30 前形成可执行志愿定稿方案，并在 2026-07-02 12:00 前只保留应急修改缓冲。

因此，稳定基座的合理完成标准应该是：

- 能保留原始结构化数据。
- 能回溯每条数据的来源页和页列。
- 能识别字段缺口、结构风险、调剂风险和家庭底线风险。
- 能筛出候选专业组，并明确哪些候选组必须人工核验。
- 不能把未闭环字段误当定稿事实。

V0 已满足前四项，并对最后一项保持硬门禁。

## 接下来怎么做

1. 停止全库无限补源。
2. 用 V0 底座先筛候选专业组。
3. 对入围专业组核完整专业组，不只看想填的 6 个专业。
4. 对入围组做字段事实核验：专业名称、专业代号、计划数、学费、选科、校区、备注限制、专业组边界、调剂范围。
5. 再结合三年投档线、515 分位次、家庭偏好、城市地区展示字段和完整专业组调剂风险生成志愿梯度方案；Round3 不按城市加分或设名额。
