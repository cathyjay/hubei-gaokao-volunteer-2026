# 《湖北招生考试》第 19 期照片 OCR 工作流

最后更新：2026-06-26

## 一、目的

第 19 期纸质版拿到后，我们需要把照片中的院校专业组、专业代号、专业名称、计划数、学费、备注等信息转成可筛选数据。

OCR 只用于加速录入和搜索，不能替代人工复核。最终志愿必须回看照片原件、官方系统和招生章程。

## 二、照片存放规则

纸质版照片默认放在本地私有目录，不提交到公开 GitHub：

```bash
mkdir -p private/raw/hubei-admission-magazine-2026-issue-19
```

建议文件命名：

```text
issue19-p076.jpg
issue19-p077.jpg
issue19-p078.jpg
```

每张照片必须满足：

- 拍完整页面，不裁掉页码、期号、院校代码、专业组、专业代号、计划数、学费和备注。
- 使用原图，不用微信压缩图。
- 页面尽量平放，避免反光、折痕、手指遮挡和明显倾斜。
- 如果一页字很密，可以整页拍一张，再把局部重点区域补拍一张。
- 先拍封面、目录、版权页或期号页，证明来源。

## 三、批量 OCR 命令

默认使用 macOS 自带 Apple Vision OCR，不需要先安装 Tesseract、PaddleOCR 或 Python 图片库。

```bash
python3 scripts/ocr_magazine_pages.py \
  --input private/raw/hubei-admission-magazine-2026-issue-19
```

默认输出到：

```text
private/ocr-runs/<时间戳>/
```

输出文件：

| 文件 | 用途 |
| --- | --- |
| `ocr.jsonl` | 每张图片完整 OCR 结果，含每行文字、置信度和位置框 |
| `text/` | 每页纯文本，方便全文搜索 |
| `manifest.csv` | 图片路径、SHA256、大小、识别行数、平均置信度 |
| `suspected_plan_lines.csv` | 疑似院校专业组、专业代号、专业名称、计划数等行 |
| `manual_review_template.csv` | 人工复核和结构化录入模板 |

## 四、复核方式

每个候选院校专业组必须完成以下复核：

1. 回看照片原件，确认 OCR 没有把数字、字母、专业名称识别错。
2. 对照 `manifest.csv` 的 SHA256，确认复核的是同一张原图。
3. 不只录入想填的 6 个专业，要录入整个院校专业组，判断是否能接受组内调剂。
4. 对学费、单科要求、外语语种、身体条件、校区、中外合作、专业备注逐项确认。
5. 最终志愿表生成前，再用湖北招生数智综合平台或纸质第 19 期逐条回看。

## 五、公开仓库规则

- `private/` 目录已被 `.gitignore` 忽略，不提交纸质版照片和整页 OCR 输出。
- 公开仓库只记录结构化后的候选数据、复核状态、来源说明和必要摘要。
- 如果需要把某一页作为证据长期保留，先确认版权和隐私风险，再决定是否只存本地。

## 六、常见风险

- `0/O`、`1/I/l`、`5/S`、`8/B` 容易混淆，院校专业组和专业代号必须人工看图。
- “物理类”“历史类”“不限”“化学”等选科字段不能只依赖 OCR。
- 备注里的“中外合作”“色盲色弱不予录取”“外语语种”等可能被识别漏掉。
- 同一学校可能有多个院校专业组，不能把不同组的专业混在一起。
