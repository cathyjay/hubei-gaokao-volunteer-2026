# 核验方法

## 一、考生基线核验

固定基线必须同时满足以下条件：

- 用户提供的成绩截图总分为 515。
- 用户提供的各科成绩相加为 515。
- 湖北官方一分一段表中，515 分累计人数为 91723。
- static section JSON 中，515 分位次区间为 `90895-91723`，累计人数为 `91723`。
- static section JSON 中，2026 年 515 分对应 2025 等位分 494、2024 等位分 497、2023 等位分 481。
- 官方 2026 控制线图片中，首选物理本科线为 435，特殊招生线为 529。

运行：

```bash
python3 scripts/verify_baseline.py
```

## 二、历年投档线核验

2025 投档线保留三层数据：

1. 湖北省教育厅官方原始图片。
2. 本地 OCR 全文。
3. 便于筛选的解析 CSV。

OCR 只是派生数据。任何从照片或图片识别出的院校代码、专业组、专业代号、计划数、学费和备注，进入候选表前都必须回看原图人工确认。

2024 和 2023 投档线保留官方 PDF、PDF 文本和解析 CSV。

最终志愿决策不能只依赖 CSV。每个入选院校专业组都必须回看对应官方原件，确认：

- 院校专业组代号
- 院校专业组名称
- 再选科目要求
- 投档最低分
- 备注，如中外合作、预科、护理类、国家专项、地方专项、民族班、定向就业等

## 三、2026 计划核验

历史投档线不能替代 2026 招生计划。每个最终候选都要核验：

- 2026 湖北招生计划或官方志愿填报系统。
- 学校 2026 本科招生章程。
- 专业组代码和专业组成是否仍然一致。
- 计划人数是否明显变化。
- 再选科目要求是否匹配“化学、地理”。
- 学费、校区、合作办学、体检限制、单科要求和备注。
- 如果 2026 计划来自第 19 期照片 OCR，必须同时核验照片原件页码、院校专业组完整范围和 OCR 输出哈希。

任何没有完成“官方计划 + 历史投档线 + 招生章程”核验的专业组，都必须保留 `needs_verification` 状态，不能进入最终表。

当前 2026 招生计划获取状态见 `docs/2026_ADMISSION_PLAN_ACQUISITION.md`。在完整计划导入前，所有院校专业组只能停留在候选池或待核验状态。

第 19 期 OCR 明细的状态流必须按以下顺序升级：

```text
OCR 初稿 / 最终可用=false
-> 原 PDF 页人工复核
-> 湖北官方平台或志愿系统核验
-> 高校官网或招生章程交叉核验
-> 家庭接受度和调剂判断
-> 三年投档稳定性判断
-> 可进入冲稳保排序
```

每次升级都要留下可追溯记录：

- 来源期号、来源 PDF SHA256、来源页码、院校代码、院校专业组代码。
- 核验日期和核验字段：专业代号、专业名称、计划数、学费、选科、校区、备注、限制条件。
- 异常说明：OCR 错字、跨页、组号变化、官网与省招办不一致、字段缺失或疑似误识别。
- 私有证据路径：截图、PDF 页图、志愿系统截图只放在 `private/`，不提交 Git。

公开 CSV 里的 `机器初判`、`综合风险等级`、`硬风险类型` 只用于安排复核顺序。命中不等于可报，未命中不等于不存在，OCR 字段不等于最终事实。

候选池页面复核包的使用方式：

- 公开元数据见 `data/working/issue19-candidate-review-page-packet.csv` 和 `data/working/issue19-candidate-review-group-page-map.csv`。
- 本地私有页图和页面 OCR 文本只用于人工回看原页，不提交 Git。
- 人工核页时要同时看候选专业组和同校相邻专业组，防止跨页、漏组、组号变化或 OCR 切组错误。
- 页面复核包生成不代表已完成核验；只有逐字段确认后，才能进入候选池 V2。

## 四、第三方数据核验

千问高考、static-data.gaokao.cn、阳光高考页面和其他工具只能作为辅助来源。

- static-data.gaokao.cn 可用于等位分交叉校验，但 2026 位次最终仍回到湖北官方一分一段。
- 千问高考可用于专业分类和候选发现，但不能直接采用其推荐概率、默认排序或预测结论。
- 阳光高考招生章程入口用于查找章程，章程内容还要逐校核对当年版本。
- 第三方来源发现的任何院校专业组，进入最终表前都必须回到湖北官方 2026 招生计划、官方投档线原件和高校招生章程复核。

## 五、提交前核验

正式提交前逐项确认：

- 登录平台网址和账号密码正确。
- 联系电话、录取通知书地址等关键信息有效。
- 45 个院校专业组顺序与最终表一致。
- 每个专业组内专业顺序一致。
- 是否服从调剂与最终表一致。
- 不愿意去的院校专业组没有被误放在前面。
- 保存、提交、确认完成，并保留最终截图或导出件到本地私密目录，不提交 Git。
